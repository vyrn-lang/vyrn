Vyrn × the host-driven event loop

examples/eventloop.vyrn keeps a counter in module state (RFC-0013): a top-level let mut hits = 0 that survives between calls. A wasm module can't block the page or suspend, so the host owns the loop — Vyrn owns the state and the logic. After _start runs main once, the timer below calls the exported onTick() handler on each fire and the button calls reset(). The count you see is live Vyrn module state. ← plain demo · extern / exports demo →
