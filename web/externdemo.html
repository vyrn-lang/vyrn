Vela × JS interop

examples/externdemo.vela declares three extern fn imports (RFC-0012) and calls them. The host page below supplies them in the vela import namespace: jsLog (a String decoded from linear memory), jsNow (returns a Float64), and jsAdd (two Int64s round- tripped through JS). The same module traps under the interpreter and the native binary — only wasm has a host. ← plain demo

Vela → JS exports (M2)



  examples/externdemo2.vela declares two export extern fns
  — a Vela function is a normal, fully-checked function that is additionally
  exported to JS on the wasm target. After _start runs
  main once, this page calls them on the live module instance:
  velaAdd(Int64, Int64) -> Int64 (BigInts round-trip) and
  greet(String) -> String (a string crosses into Vela — the
  page allocates the buffer via the module's __vela_malloc, copies
  UTF-8 + a NUL — and a built string comes back, NUL-decoded from linear memory).
