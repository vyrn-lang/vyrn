# std/i18n

std/i18n — typed translations as a library (RFC-0020 M2), built entirely on
RFC-0021 generator imports. One `gen fn`, `i18n(dir)`, reads a directory of
`<locale>.json` files at compile time (sandboxed, deterministic, cached),
parses them with the shared strict `std/json` reader, checks cross-locale
drift, compiles each ICU message into an ordinary Vyrn function body, and
RETURNS the synthesized module as source text. The compiler knows nothing about
i18n, ICU, or CLDR — everything below is plain Vyrn.

  import { i18n } from "std/i18n"
  import { t, setLocale, locale, Locale, TransKey, tCartItems } from i18n("./locales")

What the generated module contains:
  - `type Locale = | En | Uk | …`  (PascalCase of each file name)
  - `let mut currentLocale` + `setLocale(l)` / `locale()`  (module state — the
    RFC-0021 carve-out lets a SYNTHESIZED module own state)
  - `type TransKey`  — the finite validated string type of every dotted key
    (drives `t()` key-checking and LSP completion for free, RFC-0020 M1)
  - one typed function per key: `tHomeTitle() -> String`,
    `tCartItems(count: Int64) -> String`, … with args derived from the ICU
    message and a `///` doc = the source-locale message (so hover shows it)
  - `t(key: TransKey) -> String` for argument-less messages
  - `transKey(s: String) -> Option<TransKey>` — a runtime membership test over
    the finite key set, for a key that arrives as a plain `String` (config, a
    URL, user input): `Some(k)` then flows into `t(k)`, an unknown key is `None`

Inspect the synthesized module with:  vyrn emit-gen <file>

ICU subset (v1): plain text; `{name}` (String); `{n, plural, one {…} few {…}
many {…} other {…}}` (Int64, `#` = the number, `other` required); `{k,
select, a {…} other {…}}` (a finite string type of the branch names).
Apostrophe quoting: `''` = a literal apostrophe, `'{'` / `'}'` = literal
braces. Numbers/bools/null in a locale file, and nested plural/select inside a
branch, are unsupported and fail generation with a load diagnostic.

## i18n

```vyrn
fn i18n(dir: String) -> String
```

`i18n(dir)` — read every `<locale>.json` under `dir`, flatten and ICU-compile
them, check cross-locale drift, and emit the synthesized translation module.
